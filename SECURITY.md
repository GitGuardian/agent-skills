# Security Policy

## Reporting a vulnerability

Please do **not** open a public GitHub issue for security vulnerabilities.

Report security issues privately to **security@gitguardian.com**. Include enough
detail to reproduce the problem, and we will respond as quickly as we can.

This repository ships skill files that drive `ggshield` and the GitGuardian
platform. If your report concerns `ggshield` or the GitGuardian product itself
rather than these skills, the same address reaches the right team.

## Do not paste real secrets

Because these skills detect credentials, reports and issues often involve
secret-like strings. Never paste a real secret into an issue, pull request, or
email thread — redact it or replace it with an obviously fake placeholder
(for example `AKIAIOSFODNN7EXAMPLE`).
